Mission 5: Ask Me

Data: pdf-articles

Objective:

The objective of this mission is to build a Search tool that processes PDF files, performs chunking of the content, creates embeddings, stores them in MongoDB Vector Search, and allows the user to query the data. The tool should allow users to:

  1. Upload a PDF document.
  2. Perform chunking of the text content.
  3. Generate embeddings for each chunk.
  4. Store these embeddings along with sentiment analysis results in MongoDB.
  5. Allow the user to search for content.

Mission Deliverables:

Input: PDF file containing the text to analyze.
Process:
  1. Extract text content from the PDF.
  2. Chunk the text into smaller sections for analysis.
  3. Perform sentiment analysis (positive, negative, neutral) on each chunk.
  4. Generate embeddings for each chunk using a pre-trained LLM or model.
  5. Store the chunks, sentiment analysis, and embeddings in MongoDB.
Output:
  1. MongoDB database with the extracted data.
  2. The ability for users to search the content or just ask questions from the PDF, with answers based on similarity using vector search in MongoDB.
  3. Search results based on query similarity and sentiment.
